## Problem section

Generic Mac cleaners are broad. Developer machines are weird.

Your storage usually disappears into things like:

- Xcode DerivedData and Archives
- Docker layers, images, and local volumes
- `node_modules`, `.venv`, Gradle, Cargo, Go, pip, and NuGet caches
- editor and IDE caches from VS Code, Cursor, Codex, JetBrains, and Android Studio
- project artifacts like `build`, `dist`, `target`, `.next`, and `.nuxt`

DevMaid is built to surface those exact categories instead of treating them like generic clutter.

## Feature blocks

### See before you clean

Every scan result includes category, path, size, risk label, and cleanup note so you can review everything first.

### Safer cleanup flow

DevMaid quarantines selected items before restore points are lost. If you clean the wrong thing, you can undo from History.

### Made for real developer workflows

Use the native desktop app for review, or use the `devmaid` CLI for scripts, automation, and repeatable cleanup jobs.

### Local-first insight

Weekly reclaimable-space trends, low-space alerts, and cleanup history are derived from local data on your Mac.

### Exclusions that stick

Mark folders, build outputs, or projects you never want scanned again.

## Safety section

DevMaid does not hard-delete by default.

- scans are read-only
- cleanup goes through quarantine first
- restore is available from History
- danger-labeled categories require extra confirmation

## Privacy section

DevMaid works locally on your Mac.

- no external analytics in this repository version
- scan history is stored locally
- quarantine data is stored locally
